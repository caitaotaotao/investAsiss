# Stock Rationale Card

个股投资逻辑卡片的生成与持续跟踪复盘工具。

## 核心约束

**必须先读** `PHILOSOPHY.md` — 本技能所有输出必须严格遵循投资哲学约束。红灯思维：任何偏离均需人工确认方可执行。

> **⚠️ 搜索脚本硬性规则（每次搜索前必须检查）**
>
> - **脚本路径**：`python /app/custom/scripts/sogou_search.py`（禁止使用 research-workspace/scripts/ 路径）
> - **通用搜索用 `bocha`**：不含 `site:` 的查询 → `bocha "关键词" --count 10`
> - **站内搜索用 `search`**：含 `site:` 的查询 → `search "site:xxx 关键词"`
> - **禁止**对通用查询使用 `search` 子命令

## ⚠️ 上下文管理策略（防超时核心机制）

本技能在 subagent / session 中执行，context window 有限。超时的根因不是工具调用次数多，而是**原始工具输出（搜索结果、API 响应）占满 context，导致撰写卡片时 token 不够用**。

### 原则：边收集边压缩，绝不原样搬运

**1. 控制每次工具输出的体积**

| 工具 | 控制手段 | 说明 |
|------|---------|------|
| `sogou_search.py fetch` | **必须**加 `--max-chars 5000` | 原始正文默认 15000 字符，是 context 杀手 |
| `sogou_search.py bocha` | **通用搜索首选**，`--count 10`（默认） | ⚠️ 所有不含 `site:` 的查询必须用 `bocha` 子命令 |
| `sogou_search.py search` | **仅用于 `site:` 查询**，默认 1 页，禁超 3 页 | Bocha 不支持 `site:` 语法，此场景才用 `search` |
| Tushare API | 批量查询，一次调取多年多指标 | 避免拆成多次小调用（如分年度调取） |

**2. 中间研究笔记（Checkpoint 机制）**

数据收集过程中，必须分阶段将**提炼后的关键信息**写入中间笔记文件，而不是在 context 中堆积原始输出。

- **笔记文件**：`/app/custom/research-workspace/knowledge/stock-rationale-card/.scratch/{股票简称}_{代码}_{交易所}.notes.md`
- **写入时机**：每完成一个数据收集阶段后立即写入
- **写入内容**：仅写结论性数据点，不复制原始搜索结果文本
- **最终撰写时**：读取笔记文件作为主要输入来源，避免依赖超长的对话历史

**3. 工具调用后立即提炼**

每次工具调用返回结果后，**立即**在思考中提炼关键信息（3-5 条数据点），不要原样保留整段搜索结果再"稍后处理"。原始输出一旦读取提炼完毕就不会再用，但它仍占据 context 空间。

**4. 撰写卡片时读笔记、不回看原始输出**

进入撰写阶段后：
1. 读取笔记文件 `.scratch/{股票简称}_{代码}_{交易所}.notes.md`
2. 基于笔记内容填充卡片模板
3. 无需回看对话中的原始搜索结果或 API 响应

## 工作流程

### 首次建档

1. **确认股票代码** — 与用户确认目标股票，代码须含交易所后缀（沪市 `.SH`、深市 `.SZ`、北交所 `.BJ`）
2. **读取PHILOSOPHY.md** — 确认当前研究的哲学框架
3. **宽泛探测（强制）** — 先假设不知道这家公司是什么，执行 `search-for-research` 技能的**阶段零**，用归纳法探测所有可能的业务方向和新增长曲线，发现传统认知之外的线索
4. **🔖 Checkpoint 1：写入探测笔记** — 将探测发现提炼后写入 `.scratch/{股票简称}_{代码}_{交易所}.notes.md` 的"探测发现"部分（业务线清单、新线索、分析师核心观点）
5. **判断是否触发新增长曲线深挖** — 基于探测结果判断：公司是否存在新增长曲线培育期特征（新业务已启动、收入占比 <30%、处于投入期或早期放量阶段）？若是，进入步骤 5a；若否，跳至步骤 6
5a. **🔍 新增长曲线专项研究（成长股培育期必做）** — 按"新增长曲线专项研究策略"执行 ≥4 轮专项搜索，分别覆盖：① 发展方向与管理层战略表态 ② 发展动能（客户验证、技术壁垒、市场空间） ③ 成功概率（竞争格局、禀赋匹配、反面证据） ④ 市场分歧（多空逻辑、估值隐含预期）
5b. **🔖 Checkpoint 1.5：写入新增长曲线笔记** — 将专项研究结果提炼后写入笔记文件的"新增长曲线研究"部分
6. **收集信息（含迭代搜索）** — 基于探测结果，按卡片模板各模块收集数据；量/价驱动必须按**全部业务线**（含探测发现的新业务）拆分到可跟踪指标；判断公司是否产能敏感型，如是则必填产能规划模块。**搜索过程须遵循下方"研究方法论"规则。**
7. **🔖 Checkpoint 2：写入数据笔记** — 将收集到的财务数据、量/价数据、产能数据、管理层素材、风险信息提炼后写入笔记文件对应部分
8. **撰写卡片** — 读取笔记文件 `.scratch/{股票简称}_{代码}_{交易所}.notes.md`，依据PHILOSOPHY.md框架做出投资判断，按模板撰写卡片
9. **输出** — 保存至 `/app/custom/research-workspace/knowledge/stock-rationale-card/{股票简称}_{代码}_{交易所小写}.md`
10. **🗜️ 上下文压缩** — 按 AGENTS.md「Session 上下文管理协议」执行：确认卡片和笔记已写入磁盘 → 向用户汇报完成 → 调用 `session_status` 触发 context compaction，释放空间给后续任务

> **关于输出体积**：卡片中的"填写原则"引用块（`>` 标记的指引文字）**禁止出现在输出中**。表格中无实际数据的行直接删除。这两条规则可将输出体积从 ~40KB 降至 ~10-15KB，且不损失任何实质信息。

## 研究方法论：归纳法优先 + 自我修正机制

> **核心教训（杰瑞案例）**：杰瑞股份（002353）首次建档时，预设"油气装备龙头"标签，所有搜索词围绕油气设计，第三次搜索"北美 页岩气 压裂设备 海外订单"返回 0 结果后直接放弃，彻底遗漏了北美 AIDC 现场发电需求带来的燃气轮机第二增长曲线——这条新业务才是当时股价重估的核心驱动力。

### 核心原则：归纳法优先

> **改进前（演绎法，错误示范）**：
> 假设杰瑞 = 油气龙头 → 搜索油气相关内容 → 结论符合假设 → 遗漏 AIDC
>
> **改进后（归纳法，正确做法）**：
> 先假设不知道杰瑞是什么 → 宽泛探测所有业务 → 发现 AIDC 线索 → 深挖 AIDC 订单数据 → 修正结论

研究一只股票时，**先假设你不知道它是什么**，用宽泛搜索探测所有可能的方向，再深入具体领域。而不是先入为主地给它贴一个标签，然后只在标签范围内搜索。

### 搜索策略：探测-验证-风险三阶段

| 阶段 | 策略 | 具体执行 | 关键产出 |
|------|------|---------|---------|
| **探测阶段** | 归纳法宽泛搜索 | 执行 `search-for-research` 阶段零：新业务/转型探测 + 热点交叉 + 公司官网/互动易，**必须覆盖传统认知之外的可能性** | 公司全部业务方向清单（含新发现） |
| **验证阶段** | 订单+业绩双重验证 | 对探测发现的**每条业务线**分别搜索：具体订单金额、收入贡献占比、客户结构、增长趋势 | 各业务线量/价/产能数据 |
| **风险阶段** | 竞争格局+替代风险 | 新业务是否有护城河，竞争对手跟进情况，技术路线替代风险 | 风险清单 |

### 新增长曲线专项研究策略（成长股培育期必做）

当阶段零探测发现公司存在新增长曲线时，必须在验证阶段插入以下专项研究，不能仅停留在"发现了新业务"层面。

**研究目标**：回答四个核心问题——方向对不对？动能强不强？能不能成？市场怎么看？

| 研究维度 | 搜索关键词策略 | 优先信源 | 产出 |
|---------|-------------|---------|------|
| **发展方向** | "{公司} 新业务 战略规划"、"{公司} 业绩说明会 新业务"、"{公司} 投资者关系 转型" | 业绩会纪要、投资者关系活动记录、公司官网战略描述 | 新业务定位、管理层投入决心、战略优先级 |
| **发展动能** | "{公司} {新业务} 订单"、"{公司} {新业务} 客户"、"{公司} 研发投入 {新业务方向}"、"{新业务} 行业规模 渗透率" | 订单公告、行业报告、供应链调研纪要 | 客户验证进度、技术壁垒、市场空间与渗透率位置 |
| **成功概率** | "{公司} {新业务} 竞争格局"、"{新业务} 龙头 市占率"、"{公司} 管理层 转型 历史"、"{公司} 跨界" | 竞品对比研报、管理层履历、历史转型案例 | 竞争优劣势、禀赋匹配度、管理层执行力证据 |
| **市场分歧** | "site:xueqiu.com {公司} {新业务}"、"{公司} 多空 分歧"、"{公司} 估值 新业务 贡献" | 雪球讨论、卖方深度报告分歧点、互动易问答 | 多空核心逻辑、验证节点、市场隐含预期 |

**执行要点**：
- 新业务**必须独立于主业**进行搜索，禁止在主业搜索中附带了解
- 每个维度至少 1 轮专项搜索，共计 ≥4 轮搜索专门用于新增长曲线
- 管理层对新业务的表态（业绩会、互动易、路演）是最高优先级信源
- 成功概率分析需找到**至少一个反面证据**（不利因素），防止确认偏误
- 市场分歧需同时收集多方和空方观点，不能只搜支持论据

### 自我修正触发信号

| 信号 | 含义 | 应触发动作 |
|------|------|-----------|
| **搜索 0 结果** | 搜索词可能方向错误或信息源有盲区 | 换词重试 → 换引擎（Bocha）→ 换语言（英文）→ 换信源（官网/互动易），**绝不直接放弃** |
| **仅返回传统业务描述** | 搜索词不够发散，未跳出旧认知框架 | 主动搜索"新业务"、"转型"、"第二曲线" + 热点交叉词 |
| **研报观点高度一致** | 可能存在市场共识盲区 | 搜索反对意见、少数派观点、海外视角 |

### 搜索质量闭环（最多 5 轮）

使用网络搜索收集信息时，每轮搜索完成后必须对结果进行充分性评估，不满足要求时更换策略重新搜索，**最多循环 5 轮**。

**每轮搜索后执行以下检查：**

1. **阶段零完成度检查**（首轮必查）：
   | 检查项 | 判断标准 |
   |--------|---------|
   | 宽泛探测已执行？ | 3 组探测搜索（新业务/热点交叉/一手信源）是否全部完成？ |
   | 公司官网/互动易已检查？ | 是否主动检查了一手信源，而非仅依赖搜索引擎？ |
   | 发现新业务已纳入？ | 探测发现的新线索是否已作为独立维度纳入研究框架？ |

2. **逐模块覆盖度检查** — 对照卡片各模块，逐项判断：
   | 模块 | 信息充分性判断标准 |
   |------|-----------------|
   | 成长驱动（量） | 是否已按**全部**业务线（含探测发现的新业务）拆分？每条线有具体出货量/订单数据？ |
   | 成长驱动（价） | 是否有各业务线 ASP/均价数据？毛利率趋势？ |
   | 产能规划 | （产能敏感型）是否有利用率、在建项目、投产时间？ |
   | **新增长曲线** | （成长股培育期）四维度是否覆盖？① 发展方向与战略定位 ② 动能评估（≥4 项有数据） ③ 成功概率（含至少一条不利因素） ④ 市场分歧（多空双方观点） |
   | 市场叙事 | 是否有分析师路演纪要/点评？能否与驱动数据交叉验证？**新业务是否有对应叙事？** |
   | 数据支撑 | 是否有近 2-3 期财报核心数据？管理层业绩指引？ |
   | 管理层/风险 | 是否有管理层评价素材？风险事件信息？ |

3. **数量判断** — 关键模块（成长驱动、数据支撑）的有效信息条目是否 ≥3 条？若不足，标记为"信息不足"。

4. **及时性判断** — 关键数据是否为近 6 个月内？财报/纪要是否为最近一期？若过时，标记为"时效不足"。

5. **决策：继续 or 停止**
   - 所有模块均"充分" → **停止搜索**，进入撰写阶段
   - 存在"信息不足"或"时效不足"模块 → **继续下一轮**
   - 已达第 5 轮 → **强制停止**，在卡片中标注信息缺口

6. **下一轮搜索策略调整**（若继续）：
   - **更换关键词**：换用同义词、行业术语、公司别名、产品型号等
   - **切换搜索范围**：从 `site:mp.weixin.qq.com` 切换到雪球/知乎/财经媒体，或反向操作
   - **调整时间范围**：缩短到近 3 个月，或扩展到近 1 年
   - **拆分子问题**：将宽泛查询拆成更具体的子问题
   - **尝试英文搜索**：对有海外业务/对标公司的标的，补充英文查询

7. **🔖 每轮搜索后追加写入笔记**：将本轮新发现的关键数据点追加到 `.scratch/{股票简称}_{代码}_{交易所}.notes.md` 对应部分，释放 context 压力

**示例：5 轮搜索过程**
```
第 1 轮：site:mp.weixin.qq.com {公司} 路演纪要 → 找到 2 篇纪要，但缺产能数据
  → 🔖 提炼纪要核心观点写入笔记
第 2 轮：{公司} 产能利用率 扩产 投产 → 找到产能信息，但缺价格驱动数据
  → 🔖 提炼产能数据写入笔记
第 3 轮：{公司} ASP 均价 产品结构 毛利率 → 补齐价格维度
  → 🔖 提炼价格数据写入笔记
第 4 轮：{公司} 业绩会纪要 管理层指引 → 补齐管理层信息和业绩指引
  → 🔖 提炼管理层信息写入笔记
→ 覆盖度检查通过，停止搜索
```

**强制停止时的处理**：
- 在卡片对应模块中明确标注 `[信息缺口：xxx，需后续补充]`
- 在更新日志中记录搜索轮次和未覆盖内容
- 对话中向用户说明哪些信息尚未获取，建议后续补充方向

### 跟踪复盘

当用户提供新信息（财报、事件、数据）时：

1. **读取现有卡片** — 了解当前投资逻辑
2. **评估信息** — 判断是否触发逻辑变化
3. **更新卡片** — 
   - 更新 `updated:` 时间戳
   - 在更新日志追加新条目
   - 补充新数据至对应模块
   - 重新评估核心池状态（如有重大变化）
4. **🗜️ 上下文压缩** — 卡片更新完成后，按 AGENTS.md「Session 上下文管理协议」触发 context compaction

### 投资决策判断

根据PHILOSOPHY.md进行判断：

**纳入核心池**：
- 符合红利价值股框架（稳定现金流、高分红、估值修复）
- 或符合成长股框架（第二增长曲线、真增长、渗透率提升）
- 成长驱动因素清晰可追踪

**剔除/降级**：
- 核心逻辑破坏（PHILOSOPHY.md中的风险点显现）
- 数据验证与初始假设不符
- 周期位置误判

**观察维持**：
- 逻辑尚未验证，需等待关键节点确认
- 风险与机会并存，需进一步跟踪

## 质量要求

- **首次建档必须完成阶段零宽泛探测**，禁止跳过探测直接按"已知标签"构建卡片
- **业务线覆盖完整性**：卡片中的业务线必须涵盖阶段零探测发现的所有方向，不能只写传统主业
- 所有判断需引用PHILOSOPHY.md条款作为依据
- 数据需有来源（财报、研报、行业数据）
- 更新必须带时间戳，禁止覆盖历史版本
- 投资决策需明确核心逻辑，避免模糊表述
- **量/价驱动必须按业务线拆分**，禁止笼统写"收入增长"而不说明哪条线贡献、靠什么驱动
- **每条市场叙事必须有对应的驱动数据锚定**，每个关键驱动子项也应能在叙事中找到对应认知
- **产能敏感型公司（制造业、半导体、化工、新能源等）必须填写产能规划模块**，包含现有产能利用率、在建产能进度与业绩释放节奏
- **新增长曲线培育期公司必须填写 1.6 模块**，四维度（方向、动能、概率、分歧）缺一不可；成功概率分析必须包含至少一条不利因素，市场分歧必须同时呈现多空双方观点
- **搜索 0 结果时禁止直接放弃**，必须按零结果自修正规则换词/换引擎/换语言重试
- **输出卡片禁止包含模板中的填写指引文字**（`>` 引用块中的"填写原则"、"适用范围"等），只保留实际数据和结论
- **近期行情走势禁止逐日列出 K 线数据表格**（日期/开盘/最高/最低/收盘/涨跌幅/成交额），只需给出结论性点评（如"近20日跌幅14.5%，均线空头排列，关注23-24元支撑"），2-3 句话即可。逐日数据表格浪费大量 token 且无分析价值
