# Research Quality Scoring

对个股或行业的研究成果进行质量评分，分为 B / A / S 三档。评分结果用于核心池准入判断和研究深度管理。

## 核心约束

**必须先读** `PHILOSOPHY.md` — 评分标准以投资哲学为锚，不是信息量的堆砌，而是研究深度与投资逻辑完整性的综合衡量。

## 触发场景

- 用户主动要求对某标的的研究成果评分
- 更新或初始化 `core-pool-manage` 时，需先调用本技能评估研究成熟度
- 研究过程中的阶段性自检

---

## 三档评分定义

### B 档 — 基础信息搜集

**一句话**：知道这家公司是做什么的，但还没形成自己的投资逻辑。

**达标条件**（满足全部）：

| 维度 | 要求 |
|------|------|
| 公司概况 | 了解主营业务、收入结构、行业定位 |
| 财务概览 | 有近 2-3 年收入/利润/毛利率等基础数据 |
| 行业背景 | 了解所处行业的大致格局和发展阶段 |
| 信息来源 | 至少使用过 `search-for-research` 技能进行一轮搜索 |

**未达标项**：
- 尚未形成结构化的投资逻辑卡片
- 成长驱动因素（量/价/效率）未拆解
- 未做周期位置判断
- 未识别核心风险

**核心池含义**：**不具备纳入核心池条件**，仅为观察池候选。

---

### A 档 — 逻辑完整输出

**一句话**：按照投资哲学框架完成了完整的研究输出，投资逻辑清晰可追踪。

**达标条件**（在 B 档基础上，满足全部）：

| 维度 | 要求 | 对应技能/文件 |
|------|------|-------------|
| 投资逻辑卡片 | `stock-rationale-card` 六大模块填写完整 | stock-rationale-card |
| 行业定位 | 对应的 `sector-rationale-card` 已建档或已有成熟参考 | sector-rationale-card |
| 成长驱动 | 量/价/效率三维度拆解清晰，有数据支撑 | PHILOSOPHY.md §1.2 |
| 周期判断 | 明确行业与公司所处周期位置，有判断依据 | PHILOSOPHY.md §1.1 |
| 投资分类 | 明确属于红利价值股还是成长股，逻辑自洽 | PHILOSOPHY.md §2 |
| 风险识别 | 至少识别 3 个以上具体风险点，对应风险框架 | PHILOSOPHY.md §4 |
| 管理层评估 | 战略/执行/专注度/历史业绩四维度有明确评价 | stock-rationale-card §四 |
| 数据验证 | 关键结论有多源交叉验证 | PHILOSOPHY.md §3.1 |

**未达标项**：
- 没有基于研究逻辑的实际交易记录
- 没有对股价行为特征的深入理解
- 没有研究假设被市场验证/证伪的反思记录

**核心池含义**：**具备纳入核心池的基本条件**，可以小仓位参与。

---

### S 档 — 深度理解与闭环

**一句话**：不仅逻辑完整，还经历了"研究—交易—反思"的完整闭环，对公司有超越纸面的深入理解。

**达标条件**（在 A 档基础上，满足全部）：

| 维度 | 要求 | 说明 |
|------|------|------|
| **基本面深度** | 对关键财务科目变化有独立判断能力 | 不依赖研报结论，能独立拆解收入/利润变化的底层原因 |
| **预期差识别** | 能识别市场共识与自身判断的分歧点 | 符合 PHILOSOPHY.md §3.1.4 "警惕预期差"原则 |
| **股价特征理解** | 理解该股在不同市场环境下的股价行为模式 | 如：业绩驱动型/情绪驱动型/资金博弈型，波动率特征，关键价格区间 |
| **交易记录** | 有基于研究逻辑的实际交易记录（买入/卖出/持有） | 记录在 trading-workspace 中 |
| **反思记录** | 有至少一次完整的"逻辑验证—结果对比—认知更新"反思 | 包括：初始假设是什么、实际发生了什么、认知如何更新 |
| **逻辑迭代** | 投资逻辑卡片经历过至少一次基于新信息的实质性更新 | 更新日志中有非初始建档的条目 |
| **真增长/伪增长** | 对该公司的增长做过"真增长 vs 伪增长"的判断 | PHILOSOPHY.md §3.2 |

**核心池含义**：**核心池高置信度标的**，可以较大仓位参与，研究深度支撑持仓信心。

## 评分执行流程

### 1. 收集评分素材

评分前需检查以下文件是否存在及完整度：

```
必查文件（使用绝对路径）：
├── /app/custom/research-workspace/knowledge/stock-rationale-card/{代码}_{交易所小写}.md  — 个股逻辑卡片
├── /app/custom/research-workspace/knowledge/sector-rationale-card/{行业}.md  — 行业逻辑卡片
│
S 档额外检查（跨 workspace）：
├── /app/custom/trading-workspace/trading-logs/            — 交易记录
└── /app/custom/reflection-workspace/                      — 反思记录
```

### 2. 逐项评估

按当前最高可能档位的达标条件逐项检查，采用**短板原则**：任一必要条件不满足则不能达到该档位。

### 3. 写入评分结果

评分结果**直接写入被评分对象的文件**（即 `/app/custom/research-workspace/knowledge/stock-rationale-card/{代码}_{交易所小写}.md` 或 `/app/custom/research-workspace/knowledge/sector-rationale-card/{行业}.md`）。

**写入位置**：在文件末尾，作为独立的一级标题模块。如果已存在 `# 研究评分` 模块则覆盖更新。

**写入格式**：

```markdown
# 研究评分

**{B / A / S}**

评分时间：{YYYY-MM-DD}

{2-4 句话的评分支撑逻辑，说明当前档位的核心依据和升档的关键缺口，不逐条列举}
```

**示例**：

```markdown
# 研究评分

**A**

评分时间：2026-03-28

逻辑卡片六大模块完整，成长驱动拆解清晰且有财报数据支撑，周期判断引用了行业产能利用率拐点信号。升 S 档需补充实际交易记录和至少一轮"研究—交易—反思"闭环。
```

### 4. 批量评分

当需要对多篇研究内容评分时（如"给所有卡片评分"、"评估核心池全部标的"）：

**执行原则**：
- **独立评分**：逐个读取每篇 knowledge 文件，独立完成评分判断，前一个标的的评分结果不得影响后续标的的评判
- **逐个写入**：每完成一个标的的评分，立即将结果写入该文件的 `# 研究评分` 模块
- **最后汇总**：所有标的评分完成后，在对话中输出一份汇总表

**汇总输出格式**：

```
## 批量评分汇总（{YYYY-MM-DD}）

| 标的 | 类型 | 评分 | 核心短评 |
|------|------|------|---------|
| {名称}（{代码}） | 个股/行业 | B/A/S | {一句话} |
| ... | ... | ... | ... |

**分布**：S × {n}，A × {n}，B × {n}
**整体建议**：{1-2 句对研究体系现状的总体判断，如哪些方向研究较深、哪些需要补强}
```

## 与 core-pool-manage 的协同

评分结果是核心池准入的前置条件：

| 评分 | 核心池操作权限 |
|------|-------------|
| B | 仅可纳入**观察池**，不可进入核心池 |
| A | 可纳入**核心池**，建议小仓位，需持续跟踪补齐 S 档缺项 |
| S | **核心池高置信度**，仓位可适当放大 |

**降级规则**：
- 核心逻辑被证伪 → 无论当前评分，立即触发核心池复审
- 超过 3 个月未更新逻辑卡片 → 评分自动降一档
- 交易出现重大亏损且未做反思记录 → S 档降至 A 档

---

## 质量要求

- 评分必须基于实际存在的文件和记录，不得凭印象打分
- 每项达标/未达标判断需给出具体依据
- 评分结果应具备可操作性：未达标项要说清楚怎么补
- 评分不是终点，是推动研究深度持续提升的工具
