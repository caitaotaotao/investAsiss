# Sector Rationale Card

行业与产业链中观视角的投资逻辑卡片生成与跟踪复盘工具。

## 核心约束

**必须先读** `PHILOSOPHY.md` — 本技能所有输出必须严格遵循投资哲学约束。红灯思维：任何偏离均需人工确认方可执行。

## ⚠️ 上下文管理策略（防超时核心机制）

本技能在 subagent / session 中执行，context window 有限。超时的根因不是工具调用次数多，而是**原始工具输出（搜索结果、API 响应）占满 context，导致撰写卡片时 token 不够用**。

### 原则：边收集边压缩，绝不原样搬运

**1. 控制每次工具输出的体积**

| 工具 | 控制手段 | 说明 |
|------|---------|------|
| `sogou_search.py fetch` | **必须**加 `--max-chars 5000` | 原始正文默认 15000 字符，是 context 杀手 |
| `sogou_search.py search` | 默认 1 页，必要时 `--pages 2`，禁超 3 页 | 每页 ~10 条结果，2 页已足够筛选 |
| `sogou_search.py bocha` | `--count 10`（默认） | 不要设太大 |
| Tushare API | 批量查询，一次调取多年多指标 | 避免拆成多次小调用 |

**2. 中间研究笔记（Checkpoint 机制）**

数据收集过程中，必须分阶段将**提炼后的关键信息**写入中间笔记文件，而不是在 context 中堆积原始输出。

- **笔记文件**：`/app/custom/research-workspace/knowledge/sector-rationale-card/.scratch/{行业名称}.notes.md`
- **写入时机**：每完成一个数据收集阶段后立即写入
- **写入内容**：仅写结论性数据点，不复制原始搜索结果文本
- **最终撰写时**：读取笔记文件作为主要输入来源，避免依赖超长的对话历史

笔记文件结构：


**3. 工具调用后立即提炼**

每次工具调用返回结果后，**立即**在思考中提炼关键信息（3-5 条数据点），不要原样保留整段搜索结果再"稍后处理"。原始输出一旦读取提炼完毕就不会再用，但它仍占据 context 空间。

**4. 撰写卡片时读笔记、不回看原始输出**

进入撰写阶段后：
1. 读取笔记文件 `.scratch/{行业名称}.notes.md`
2. 基于笔记内容填充卡片模板
3. 无需回看对话中的原始搜索结果或 API 响应

## 工作流程

### 首次建档

1. **确认行业** — 与用户确认目标行业/产业链，需要了解清楚行业/产业链具体边界
2. **读取PHILOSOPHY.md** — 确认中观研究的哲学框架
3. **产业研究（含迭代搜索）** — 按卡片模板8大模块收集数据。**搜索过程须遵循下方"搜索质量闭环"规则。**
4. **🔖 Checkpoint：写入研究笔记** — 每完成一轮搜索后，将提炼后的关键信息写入 `.scratch/{行业名称}.notes.md` 对应部分（产业链结构、驱动因素、竞争格局、景气数据等），释放 context 压力
5. **撰写卡片** — 读取笔记文件 `.scratch/{行业名称}.notes.md`，依据PHILOSOPHY.md框架做出行业周期判断，按模板撰写卡片
6. **输出** — 保存至 `/app/custom/research-workspace/knowledge/sector-rationale-card/{行业名}.md`
7. **🗜️ 上下文压缩** — 按 AGENTS.md「Session 上下文管理协议」执行：确认卡片和笔记已写入磁盘 → 向用户汇报完成 → 调用 `session_status` 触发 context compaction，释放空间给后续任务

> **关于输出体积**：卡片中的"核心原则"、"说明"等指引文字**禁止出现在输出中**。行业特性景气度模板（6.2 节）中仅输出与目标行业匹配的模板，其余行业模板不出现在输出卡片中。表格中无实际数据的行直接删除。

### 搜索质量闭环（最多 5 轮）

使用网络搜索收集信息时，每轮搜索完成后必须对结果进行充分性评估，不满足要求时更换策略重新搜索，**最多循环 5 轮**。

**每轮搜索后执行以下检查：**

1. **逐模块覆盖度检查** — 对照卡片八大模块，逐项判断：
   | 模块 | 信息充分性判断标准 |
   |------|-----------------|
   | 产业链结构 | 是否已梳理上中下游环节？核心壁垒和价值分布是否清晰？ |
   | 核心驱动因素 | 量/价/效率/政策四维度是否各有数据支撑？ |
   | 竞争格局 | CR5/龙头份额是否有具体数据？格局演变方向是否有依据？ |
   | 周期位置 | 是否有景气度指标（领先/同步/滞后）支撑周期判断？ |
   | 市场叙事 | 是否有分析师行业观点、政策催化信息？ |
   | 重点数据跟踪 | 景气度指标体系是否已建立？是否有高频数据？ |
   | 投资逻辑 | 看好/风险逻辑是否有足够数据交叉验证？ |
   | 相关公司池 | 是否已识别各环节代表公司？ |

2. **数量判断** — 关键模块（驱动因素、周期位置、景气指标）有效信息条目是否 ≥3 条？若不足，标记为"信息不足"。

3. **及时性判断** — 行业数据是否为近 6 个月内？景气高频指标是否为近 1-3 个月？若过时，标记为"时效不足"。

4. **决策：继续 or 停止**
   - 所有模块均"充分" → **停止搜索**，进入撰写阶段
   - 存在"信息不足"或"时效不足"模块 → **继续下一轮**
   - 已达第 5 轮 → **强制停止**，在卡片中标注信息缺口

5. **下一轮搜索策略调整**（若继续）：
   - **更换关键词**：换用行业别名、子领域名称、上下游环节名
   - **切换搜索范围**：从公众号切到行业协会官网/雪球/知乎，或反向操作
   - **调整时间范围**：缩短到近 3 个月，或扩展到近 1 年
   - **拆分子问题**：从"行业整体"拆成"上游原料"/"中游制造"/"下游应用"分别搜索
   - **尝试英文搜索**：对有全球可比口径的行业，补充海外行业数据和对标分析

6. **🔖 每轮搜索后追加写入笔记**：将本轮新发现的关键数据点追加到 `.scratch/{行业名称}.notes.md` 对应部分，释放 context 压力

### 跟踪复盘

当用户提供新信息（政策、数据、事件、公司财报中的行业信息）时：

1. **读取现有卡片** — 了解当前行业逻辑
2. **评估信息** — 判断是否触发逻辑或周期变化
3. **更新卡片** — 
   - 更新 `updated:` 时间戳
   - 在更新日志追加新条目
   - 补充新数据至对应模块
   - 重新评估周期位置（如有重大变化）
4. **🗜️ 上下文压缩** — 卡片更新完成后，按 AGENTS.md「Session 上下文管理协议」触发 context compaction

### 周期判断框架（基于PHILOSOPHY.md）

**复苏期**：需求边际改善、产能利用率触底、行业盈利能力开始修复
**成长期**：渗透率快速提升、量价齐升、竞争格局尚未固化
**成熟期**：增速放缓、竞争格局稳定、龙头市占率稳定
**衰退期**：需求萎缩、产能过剩、价格战频发

## 质量要求

- 所有判断需引用PHILOSOPHY.md条款作为依据
- 行业周期判断需有具体数据支撑
- 产业链价值分布需明确上中下游的壁垒差异
- 更新必须带时间戳，禁止覆盖历史版本
- **输出卡片禁止包含模板中的写作指引文字**（"核心原则"、"说明"等引用块），只保留实际数据和结论
- **行业特性景气度模板（6.2 节）仅输出目标行业对应的模板**，其余行业模板不出现在输出中
