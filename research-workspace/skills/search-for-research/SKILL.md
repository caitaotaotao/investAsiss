# Search for Research

投资研究领域的深度搜索与多源信息收集技能。采用迭代式深度搜索策略，从宏观、中观、微观三个层次系统性收集投研所需信息，为个股卡片和行业卡片提供高质量数据输入。

## 核心约束

**必须先读** `PHILOSOPHY.md` — 搜索方向和信息筛选必须围绕投资哲学框架，避免收集与投资逻辑无关的噪音信息。

## 设计理念

参考业界高星开源项目的核心架构思想：

- **迭代式深度搜索**：借鉴 dzhng/deep-research 的 depth/breadth 递归搜索模式，每轮搜索基于上一轮结果动态调整方向
- **任务分解与自校验**：借鉴 virattt/dexter 的 task planning + self-validation 模式，将复杂研究问题拆解为结构化子任务，每步校验信息质量
- **多源数据工具协同**：借鉴 schnetzlerjoe/hermes 的多 agent 多工具协作模式，从财报、宏观、行情、新闻等多维度交叉验证

## 搜索参数配置

每次深度搜索任务启动前，根据研究目标确定参数：

| 参数 | 说明 | 推荐值 | 适用场景 |
|------|------|--------|---------|
| **breadth（广度）** | 每轮并发搜索的查询数量 | 3-6 | 广度越大覆盖面越全，但耗时越长 |
| **depth（深度）** | 递归搜索的最大层数 | 2-4 | 深度越大挖掘越深，适合复杂主题 |
| **focus（聚焦维度）** | 搜索聚焦的分析层次 | macro/sector/company | 决定搜索的起始方向 |

### 预设搜索模板

| 模板名 | breadth | depth | focus | 用途 |
|--------|---------|-------|-------|------|
| **快速扫描** | 3 | 1 | 单一维度 | 快速了解某个话题或验证单一信息点 |
| **标准调研** | 4 | 2 | company 或 sector | 为建档提供基础信息 |
| **深度研究** | 6 | 3 | 全维度 | 重大投资决策前的全面调研 |
| **事件追踪** | 3 | 3 | 事件链 | 追踪某一事件的前因后果和影响链 |

## 搜索工具：双引擎检索脚本

本技能使用 `/app/custom/scripts/sogou_search.py` 作为核心搜索工具，支持**搜狗搜索**和 **Bocha WebSearch API** 双引擎。

- **搜狗搜索（`search`）**：基于爬虫，免费，支持 `site:` 语法精准定位，但结果数量和质量可能不稳定
- **Bocha WebSearch API（`bocha`）**：付费 API，结果质量更高、结构化更好、带时间戳，作为搜狗搜索的**补充和降级方案**

### 搜索引擎选择策略

```
默认使用搜狗搜索（免费、支持 site: 语法）
        │
        ▼
  搜狗结果是否满足要求？
  ├── 结果数量 ≥ 5 条 且 内容相关性高 → 继续使用搜狗
  └── 结果数量 < 5 条 或 大量无关结果 或 被反爬拦截 → 切换 Bocha API
```

**触发切换 Bocha 的条件**（满足任一即切换）：
- 搜狗返回结果数量 < 5 条
- 搜狗返回结果中，与查询相关的有效结果 < 3 条
- 搜狗被反爬机制拦截（返回 0 结果或异常页面）
- 需要带时间戳的搜索结果来判断信息时效性
- 搜狗 `site:` 语法搜索后结果为空，需要更广泛的检索

### 前置依赖

```bash
pip install requests beautifulsoup4
```

环境变量：`BOCHA_API_KEY`（使用 bocha 命令时必需）

### 命令一览

| 功能 | 命令 | 说明 |
|------|------|------|
| **搜狗搜索** | `python /app/custom/scripts/sogou_search.py search "关键词"` | 默认引擎，返回结构化结果列表 |
| 搜狗多页 | `python /app/custom/scripts/sogou_search.py search "关键词" --pages 3` | 搜索多页，每页约 10 条结果 |
| 搜狗快速 | `python /app/custom/scripts/sogou_search.py search "关键词" --no-resolve` | 跳过重定向解析，速度更快 |
| **Bocha 搜索** | `python /app/custom/scripts/sogou_search.py bocha "关键词"` | Bocha API，结果质量更高，带时间戳 |
| Bocha 多条 | `python /app/custom/scripts/sogou_search.py bocha "关键词" --count 20` | 指定返回条数（最大 50） |
| JSON 输出 | `python /app/custom/scripts/sogou_search.py search "关键词" --json` | 两个引擎均支持 `--json` |
| **抓取正文** | `python /app/custom/scripts/sogou_search.py fetch "https://..."` | 提取目标 URL 的可读正文内容 |
| 限制字数 | `python /app/custom/scripts/sogou_search.py fetch "https://..." --max-chars 8000` | 限制返回正文字符数 |

### AI 搜索操作流程

执行搜索任务时，按以下步骤使用脚本：

1. **执行搜索（优先搜狗）** — 根据子问题生成查询词，调用 `search` 命令获取结果列表

   ```bash
   python /app/custom/scripts/sogou_search.py search "贵州茅台 2025年报 业绩" --pages 2
   ```

2. **评估搜索结果质量** — 检查返回结果数量和相关性：
   - 结果 ≥ 5 条且相关 → 继续步骤 3
   - 结果不足或不相关 → **切换 Bocha 重新搜索**：
     ```bash
     python /app/custom/scripts/sogou_search.py bocha "贵州茅台 2025年报 业绩" --count 15
     ```

3. **阅读结果列表** — 浏览返回的标题、来源域名和摘要，筛选出符合信息源白名单的高质量结果

4. **深入阅读** — 对有价值的链接使用 `fetch` 命令获取全文内容

   ```bash
   python /app/custom/scripts/sogou_search.py fetch "https://xueqiu.com/xxx/article"
   ```

5. **站内搜索** — 配合 `site:` 语法精准定位白名单信息源（仅搜狗支持）

   ```bash
   python /app/custom/scripts/sogou_search.py search "site:xueqiu.com 宁德时代 深度分析"
   python /app/custom/scripts/sogou_search.py search "site:mp.weixin.qq.com 半导体 产业链"
   ```

   若 `site:` 搜索结果为空，用 Bocha 做更广泛的检索：
   ```bash
   python /app/custom/scripts/sogou_search.py bocha "宁德时代 深度分析 雪球" --count 10
   ```

6. **排除黑名单** — 搜索时主动排除低质量源

   ```bash
   python /app/custom/scripts/sogou_search.py search "比亚迪 销量 -site:guba.eastmoney.com -site:baijiahao.baidu.com"
   ```

### 注意事项

- **请求频率**：搜狗脚本内置随机延迟（1-2.5 秒/页），避免触发反爬。单次搜索建议不超过 5 页
- **来源校验**：结果中的 `source` 字段即为域名，用于快速判断是否命中黑名单
- **URL 解析**：搜狗默认会解析重定向链接为真实 URL；加 `--no-resolve` 可跳过以提升速度
- **正文提取**：`fetch` 自动去除导航、广告等非正文内容，默认最多返回 15000 字符
- **Bocha 优势**：结果带 `date` 字段（爬取时间），便于判断信息时效性；结果质量整体优于搜狗爬虫
- **成本控制**：Bocha 为付费 API，优先用搜狗，仅在搜狗结果不足时切换

## 搜索工作流程

### 阶段一：问题分解（Task Decomposition）

将用户的研究问题拆解为结构化子问题，明确搜索意图：

1. **识别研究目标类型**
   - 个股研究 → 对接 `stock-rationale-card` 的信息需求
   - 行业研究 → 对接 `sector-rationale-card` 的信息需求
   - 主题研究 → 围绕特定投资主题（如"国产替代"、"AI算力"）
   - 事件研究 → 追踪某一事件对投资逻辑的影响

2. **生成子问题清单**

   以个股研究为例，自动拆解为：

   ```
   [核心问题] 公司XXX是否值得纳入核心池？
   ├── [子问题1] 公司主营业务和核心竞争力是什么？
   ├── [子问题2] 成长驱动因素（量/价/效率）分别是什么？
   ├── [子问题3] 所处行业的周期位置在哪里？
   ├── [子问题4] 管理层的战略能力和执行力如何？
   ├── [子问题5] 近期有哪些关键催化剂或风险事件？
   └── [子问题6] 当前估值水平是否合理？
   ```

3. **为每个子问题生成初始搜索查询**
   - 每个子问题生成 2-3 个不同角度的搜索查询
   - 查询语言：优先中文（A股标的），辅以英文（全球视野）
   - 查询策略：混合使用精确关键词和自然语言问题

### 阶段二：迭代搜索（Iterative Deep Search）

采用广度优先 + 深度递归的搜索策略：

```
第 1 层（广度搜索）
├── 查询1 → 结果 → 提取关键发现 + 新方向
├── 查询2 → 结果 → 提取关键发现 + 新方向
├── 查询3 → 结果 → 提取关键发现 + 新方向
└── 汇总本层发现 → 生成下一层查询

第 2 层（深度递归）
├── 基于第1层发现的新查询1 → 更深入的结果
├── 基于第1层发现的新查询2 → 更深入的结果
└── 汇总 → 判断是否需要继续深入

第 N 层（达到 depth 上限或信息饱和时停止）
└── 最终汇总所有发现
```

**每轮搜索的处理步骤**：

1. **执行搜索** — 使用 `python /app/custom/scripts/sogou_search.py search` 执行查询（搜狗结果不足时切换 `bocha` 命令），必要时用 `fetch` 深入阅读
2. **提取关键发现（Learnings）** — 从搜索结果中提炼与投资逻辑相关的关键信息
3. **识别新方向（Directions）** — 发现需要进一步探索的线索
4. **信息质量评估** — 判断信息来源可靠性、时效性、与投资逻辑的相关性
5. **决策：继续/停止** — 信息是否饱和？是否还有关键缺口？

**搜索终止条件**：
- 达到 depth 上限
- 信息饱和（连续两轮无实质性新发现）
- 子问题已全部覆盖且信息充分

### 阶段三：信息整合与校验（Synthesis & Validation）

1. **去重与合并** — 去除重复信息，合并同一主题的多源发现
2. **交叉验证** — 对关键数据点进行多源交叉确认（符合 PHILOSOPHY.md 3.1 多重交叉验证原则）
3. **标注置信度** — 对每条关键发现标注信息质量等级
4. **识别信息缺口** — 明确哪些关键问题尚未获得充分信息
5. **生成结构化输出** — 按标准模板输出搜索结果

### 搜索时的源控制技巧

执行搜索时，主动使用以下策略提升结果质量：

1. **站内搜索限定**：对白名单源使用 `site:` 语法精准搜索
   - `site:xueqiu.com {公司名} 深度分析`
   - `site:mp.weixin.qq.com {行业} 产业链`
   - `site:zhihu.com {技术} 行业前景`

2. **排除黑名单**：在通用搜索中主动排除低质量源
   - `-site:guba.eastmoney.com -site:baijiahao.baidu.com -site:tieba.baidu.com`

3. **优先级搜索顺序**：每个子问题按以下顺序搜索
   - **第一轮：微信公众号纪要/点评**（路演纪要、业绩会纪要、分析师点评、调研纪要）→ 信息密度最高，优先获取
   - 第二轮：P0-P1 来源（官方公告、财报原文、权威行业数据）→ 交叉验证纪要中的关键数据
   - 第三轮：雪球/知乎 深度分析 + 微信公众号行业研究 → 补充分析框架和多元视角
   - 第四轮：财经媒体报道补充
   - 仅在信息不足时才扩展到其他来源

4. **结果标注**：每条搜索结果必须标注实际来源域名，便于审核是否命中黑名单

### 使用规则

- 关键投资决策必须有 P0-P1 级信息支撑
- P2 级信息可作为分析框架和逻辑参考
- P3 级信息用于补充背景和多元视角
- P4 级信息仅作为情绪和预期差的辅助判断，且必须来自白名单来源
- 所有引用信息必须标注来源和时间
- **命中黑名单的内容一律不得出现在最终输出中**

## 搜索结果输出规范

搜索结果**不单独持久化存储**，而是直接在对话中以结构化格式呈现，供用户判断后再由下游技能（`stock-rationale-card`、`sector-rationale-card`）吸收到对应的 knowledge 文件中。

### 输出格式

搜索完成后，在对话中按以下结构输出：

**1. 搜索概述**
- 研究问题、搜索范围（宏观/中观/微观）、实际搜索轮次
- 信息完整度评估（高/中/低）及缺口说明

**2. 核心发现**（按重要性排序）
- 每条发现标注置信度（高/中/低）、信息来源及时间
- 与 PHILOSOPHY.md 的关联：周期位置判断、成长驱动验证、风险因素更新

**3. 详细发现**
- 按宏观/行业/公司分层组织

**4. 信息质量评估**
- 关键信息点的来源等级（P0-P4）、时效性、是否交叉验证

**5. 信息缺口与后续建议**
- 未覆盖的关键问题、建议的后续行动

**6. 引用来源**
- 所有引用的来源标题、URL、日期

## 特殊搜索场景

### 事件驱动搜索

当突发事件（政策变化、行业事故、公司公告等）发生时：

1. **事件本身** — 搜索事件的完整信息和官方表态
2. **历史对标** — 搜索类似历史事件及其后续影响
3. **影响链条** — 沿产业链搜索上下游受影响环节
4. **市场反应** — 搜索市场即时反应和机构观点
5. **持续跟踪** — 标记需要持续关注的后续节点

### 财报季搜索

财报发布期间的高效搜索策略：

1. **业绩速读** — 搜索收入/利润/现金流核心指标 vs 市场预期
2. **边际变化** — 搜索环比/同比的关键变化及管理层解释
3. **指引更新** — 搜索管理层对未来季度的业绩指引
4. **同行对比** — 搜索同行业已披露财报的共性趋势
5. **预期差定位** — 结合搜索结果判断实际业绩与市场预期的差距

### 主题投资搜索

围绕特定投资主题（如"AI应用落地"、"出海"）的搜索：

1. **主题定义** — 明确主题边界和核心逻辑
2. **产业链映射** — 搜索主题涉及的完整产业链图谱
3. **受益排序** — 搜索不同环节的受益程度和确定性
4. **催化剂日历** — 搜索未来可能兑现的关键节点
5. **风险评估** — 搜索主题可能被证伪的条件

## 质量要求

- **信息时效性**：优先使用近 3 个月内的信息，超过 6 个月的数据需标注"历史数据"
- **来源标注**：每条关键信息必须标注来源和日期，无来源信息不得作为决策依据
- **交叉验证**：关键数据点至少需要 2 个独立来源确认
- **投资相关性**：所有搜索必须服务于 PHILOSOPHY.md 框架下的投资逻辑判断，避免收集无关信息
- **增量价值**：每轮搜索应产生增量信息，避免重复搜索已知内容
- **偏见识别**：标注信息来源的潜在立场偏差（如卖方研报的利益冲突、媒体的流量导向）
