# Core Pool Manage

股票核心池的建立、更新与日常维护。核心池是从全部研究标的中筛选出的高置信度投资候选集合。

## 核心约束

**必须先读** `PHILOSOPHY.md` — 核心池的准入标准以投资哲学为锚。红灯思维：任何偏离均需人工确认方可执行。

---

## 池规模与结构约束

| 约束项 | 规则 |
|--------|------|
| **总规模** | 控制在 **500 只**左右（允许 ±10% 浮动，即 450-550） |
| **成长股优先** | 超出 500 只时，优先保留科技、医药等成长股（PHILOSOPHY.md §2.2） |
| **红利价值股下限** | 红利价值股数量 **不得低于 50 只**（PHILOSOPHY.md §2.1） |

**投资分类判断依据**：以每只股票的 `stock-rationale-card` 中"投资决策 → 决策依据"字段为准，该字段应明确标注属于红利价值股还是成长股。

---

## 核心池文件

- **路径**：`./knowledge/core_stock_pool.xlsx`
- **列定义**：

| 列名 | 类型 | 说明                                     |
|------|------|----------------------------------------|
| code | string | 股票代码，含交易所后缀（如 `600519_sh`、`000001_sz`） |
| short_name | string | 股票简称（如 `贵州茅台`、`平安银行`）                  |
| in_pool_date | date | 纳入核心池日期（YYYY-MM-DD）                    |
| out_pool_date | date | 移出核心池日期（YYYY-MM-DD），在池中时为空             |
| update_date | date | 最近一次评估/更新日期（YYYY-MM-DD）                |

**读写规则**：
- 文件不存在时自动创建
- 纳入新股票：追加行，填写 `code`、`short_name`、`in_pool_date`、`update_date`，`out_pool_date` 留空
- 移出股票：**不删除行**，填写 `out_pool_date`，保留历史记录
- 重新纳入：追加新行（不复用旧行），生成新的 `in_pool_date`

## 工作流程

### 流程一：初始化核心池

用户首次要求建立核心池时执行。

**步骤**：

1. **扫描全部卡片**
   读取 `./knowledge/stock-rationale-card/` 目录下所有 `.md` 文件。

2. **检查研究评分**
   对每张卡片检查 `# 研究评分` 模块：
   - **无评分** → 调用 `research-quality-scoring` 进行评分
   - **评分存在但超过 1 个月未更新** → 调用 `research-quality-scoring` 重新评分
   - **评分存在且在 1 个月内** → 直接采用

3. **筛选准入**
   仅 **A 档和 S 档** 股票进入候选集。B 档不纳入。

4. **规模调控**
   如候选集超过 500 只：
   - 先确保红利价值股 ≥ 50 只
   - 剩余名额按以下优先级排序截取：
     - S 档优先于 A 档
     - 同档位内，科技/医药成长股优先
   
   如候选集不足 500 只：正常纳入，无需补齐。

5. **写入 Excel**
   将最终名单写入 `./knowledge/core_stock_pool.xlsx`。

6. **更新卡片状态**
   将纳入核心池的股票，在其 `stock-rationale-card` 文件头部的 `core-pool` 字段更新为 `纳入`。

7. **对话汇总**
   在对话中输出初始化结果：

   ```
   ## 核心池初始化完成（{YYYY-MM-DD}）

   - 纳入总数：{n} 只
   - 成长股：{n} 只（科技 {n} / 医药 {n} / 其他 {n}）
   - 红利价值股：{n} 只
   - 评分分布：S × {n}，A × {n}
   - 本次新评分/重新评分：{n} 只
   ```

### 流程二：更新核心池

用户要求更新核心池时执行。

**步骤**：

1. **读取现有核心池**
   读取 `./knowledge/core_stock_pool.xlsx`，获取当前在池股票清单（`out_pool_date` 为空的行）。

2. **逐只复审在池股票**
   对每只在池股票：
   - 读取其 `stock-rationale-card`，检查研究评分时效性（超过 1 个月则重新评分）
   - 检查是否仍满足核心池准入条件：
     - 评分是否降至 B 档 → **移出**
     - 卡片的 `core-pool` 字段是否已被手动改为 `剔除` → **移出**
     - 逻辑卡片超过 3 个月未更新（`updated` 字段） → **标记待审**，在对话中提醒用户确认

3. **扫描新候选**
   扫描 `./knowledge/stock-rationale-card/` 中不在核心池的卡片：
   - 评分为 A 或 S 档 → 纳入候选
   - 检查评分时效性，必要时重新评分

4. **规模调控**
   合并现有在池股票（移出的除外）与新候选，执行与初始化相同的规模调控逻辑。

5. **执行变更**
   - 移出的股票：在 Excel 中填写 `out_pool_date`，更新卡片 `core-pool` 为 `剔除`
   - 新纳入的股票：在 Excel 中追加行，更新卡片 `core-pool` 为 `纳入`
   - 所有在池股票：更新 Excel 中的 `update_date`

6. **对话汇总**

   ```
   ## 核心池更新完成（{YYYY-MM-DD}）

   - 当前池内：{n} 只（成长 {n} / 红利价值 {n}）
   - 本次新纳入：{n} 只
   - 本次移出：{n} 只
   - 评分分布：S × {n}，A × {n}

   ### 新纳入
   | 代码 | 简称 | 评分 | 投资分类 |
   |------|------|------|---------|
   | ... | ... | ... | ... |

   ### 移出
   | 代码 | 简称 | 移出原因 |
   |------|------|---------|
   | ... | ... | {评分降档/逻辑证伪/手动剔除} |

   ### 待审（逻辑卡片长期未更新）
   | 代码 | 简称 | 上次更新 | 建议 |
   |------|------|---------|------|
   | ... | ... | ... | 更新卡片或确认移出 |
   ```

## 与其他技能的协同

| 协同技能 | 交互方式 |
|---------|---------|
| `research-quality-scoring` | 核心池初始化/更新时，自动调用进行评分或刷新过期评分 |
| `stock-rationale-card` | 读取卡片内容和评分，更新 `core-pool` 字段 |
| `sector-rationale-card` | 行业卡片为个股投资分类（成长/红利价值）提供上下文参考 |
| `search-for-research` | 更新核心池时，若发现信息过时，可建议用户调用搜索补充 |

---

## 质量要求

- 核心池变更（纳入/移出）必须有明确的评分和逻辑依据，不得主观臆断
- 移出操作保留历史记录，不删除 Excel 行
- 规模调控时的取舍决策需在对话汇总中说明理由
- 红利价值股 50 只下限为硬约束，任何情况下不得突破
