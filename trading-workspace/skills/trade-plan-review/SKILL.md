# Trade Plan Review

审查用户提出的交易计划，评估合理性，确认后记录至交易日志。

## 核心约束

- **必须先读** `../research-workspace/PHILOSOPHY.md` — 交易计划的评估必须基于投资哲学框架
- 审查是**辅助判断**，最终交易决策权在用户手中
- 不可在用户未明确确认的情况下执行任何记录操作

---

## 触发场景

- 用户提出买入/卖出/调仓计划
- 用户要求评估某笔交易是否合理
- 用户分享交易想法，寻求第二意见

---

## 审查框架

### 1. 提取交易要素

从用户输入中识别以下字段（缺失项须主动询问）：

| 要素 | 说明 | 必填 |
|------|------|------|
| 标的 | 股票代码 + 简称 | 是 |
| 方向 | 买入 / 卖出 / 加仓 / 减仓 | 是 |
| 计划价格/价格区间 | 目标建仓或出场价格 | 是 |
| 计划数量或金额 | 拟投入资金或股数 | 是 |
| 交易逻辑 | 为什么做这笔交易 | 是 |
| 止损位 | 预设的止损价格或条件 | 建议 |
| 目标位 | 预期的止盈价格或条件 | 建议 |
| 持有周期预期 | 短线 / 波段 / 中长线 | 建议 |

### 2. 多维度审查

审查从以下 5 个维度进行，每个维度给出 **通过 / 提醒 / 警告** 标记：

#### 2.1 研究支撑度

- 该标的是否有对应的 rationale-card？研究评分档位？
- 是否在核心池内？
- 交易逻辑与 rationale-card 中的投资逻辑是否一致？
- **警告**：无研究基础的标的建仓

#### 2.2 仓位合理性

- 读取当前持仓（`./holdings/holdings.xlsx`）
- 计算该笔交易后的预期持仓权重
- 单标的权重是否过高（>15% 需提醒，>25% 需警告）
- 同行业/同主题集中度是否过高

#### 2.3 风险收益比

- 止损位与目标位是否设定？
- 计算潜在风险收益比（目标位盈利 / 止损位亏损）
- **提醒**：风险收益比 < 2:1
- **警告**：未设定止损位

#### 2.4 周期与时机

- 当前市场环境评估（如用户有提供相关信息）
- 标的所处的行业周期位置（参考 rationale-card）
- 是否存在追高或抄底过早的风险

#### 2.5 交易纪律

- 是否存在频繁交易倾向（参考近期 trading-logs）
- 是否与此前的交易计划矛盾
- 是否存在情绪化交易的特征（如急于加仓刚亏损的标的）

### 3. 输出审查结果

在对话中以结构化格式呈现：

```
## 交易计划审查

**标的**：{简称}（{代码}）
**方向**：{买入/卖出}  |  **计划价格**：{价格}  |  **数量/金额**：{数量}

### 审查结论

| 维度 | 判定 | 说明 |
|------|------|------|
| 研究支撑 | ✅ 通过 / ⚠️ 提醒 / 🚫 警告 | {一句话} |
| 仓位合理 | ✅ / ⚠️ / 🚫 | {一句话} |
| 风险收益 | ✅ / ⚠️ / 🚫 | {一句话} |
| 周期时机 | ✅ / ⚠️ / 🚫 | {一句话} |
| 交易纪律 | ✅ / ⚠️ / 🚫 | {一句话} |

### 综合意见
{2-4 句话的综合评价，指出最大风险点和建议}

---
**是否确认执行该交易计划？**（回复"确认"将记录至交易日志）
```

### 4. 用户确认后执行

用户回复确认后：

1. **调用 `trading-log` 技能**：将交易计划写入 `./trading-logs/trading_log.xlsx`，状态为 `open`
2. **调用 `holdings-manage` 技能**：更新持仓信息
3. 在对话中确认记录完成

**如果用户决定不执行**：在对话中确认取消，不做任何记录。

---

## 审查原则

- **不做决策，做辅助**：审查结果是参考意见，不替用户做最终决定
- **直言不讳**：发现问题直接指出，不和稀泥
- **数据说话**：尽量用已有的 rationale-card、持仓数据、历史交易记录来支撑判断
- **尊重用户**：即使审查结论为多个警告，用户坚持执行仍应尊重并记录
