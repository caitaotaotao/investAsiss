# Trade Review

对指定区间内的已平仓交易进行复盘分析，输出量化指标与行为评价。

## 核心约束

- **必须先读** `/app/custom/research-workspace/PHILOSOPHY.md` — 行为评价须以投资哲学为标尺
- 复盘基于**已平仓**（`status = closed`）交易记录，未平仓交易不纳入胜率等统计
- 复盘结果**仅在对话中输出**，不持久化存储（用户可自行要求保存）

---

## 触发场景

- 用户指定时间区间要求复盘（如"复盘三月份的交易"）
- 用户要求总结某类标的的交易表现（如"复盘科技股的交易"）
- 定期回顾（如月度/季度交易总结）

---

## 执行流程

### 1. 确定复盘范围

从用户输入中提取：
- **时间区间**：起止日期（必填，若用户说"这个月"则解析为具体日期）
- **筛选条件**（可选）：特定标的、行业、方向等

### 2. 读取交易数据

从 `/app/custom/trading-workspace/trading-logs/trading_log.xlsx` 中筛选满足条件的 `status = closed` 记录。

若区间内无已平仓交易，提示用户并展示该区间内的未平仓交易概况。

### 3. 计算量化指标

#### 核心指标

| 指标 | 计算方式 |
|------|---------|
| **总交易笔数** | 区间内已平仓交易条数 |
| **盈利笔数 / 亏损笔数** | pnl > 0 为盈利，pnl ≤ 0 为亏损 |
| **胜率** | 盈利笔数 / 总交易笔数 |
| **总盈亏** | 所有交易 pnl 之和 |
| **平均盈利** | 盈利交易的 pnl 均值 |
| **平均亏损** | 亏损交易的 pnl 均值（绝对值） |
| **赔率（盈亏比）** | 平均盈利 / 平均亏损 |
| **期望值** | 胜率 × 平均盈利 - (1 - 胜率) × 平均亏损 |
| **最大单笔盈利** | max(pnl) |
| **最大单笔亏损** | min(pnl) |
| **平均持有天数** | holding_days 均值 |
| **盈利交易平均持有天数** | 盈利交易的 holding_days 均值 |
| **亏损交易平均持有天数** | 亏损交易的 holding_days 均值 |

### 4. 交易行为评价

基于量化指标和交易明细，从以下维度进行定性分析：

#### 4.1 纪律执行

- 是否按预设止损位执行？（对比 stop_loss 与 close_price）
- 是否存在未设止损的交易？
- 止盈是否过早或贪婪？（对比 target_price 与 close_price）

#### 4.2 仓位管理

- 单笔交易金额是否合理分布？
- 是否存在某笔交易金额远超平均水平

#### 4.3 交易频率

- 区间内的交易频率是否合理？
- 短持有天数（<3 天）的交易占比
- 是否存在同一标的反复交易的情况

#### 4.4 研究-交易一致性

- 交易逻辑（trade_rationale）是否与 rationale-card 的投资逻辑一致？
- 是否存在"冲动交易"（无研究支撑的标的）

#### 4.5 盈亏结构

- 盈利是集中在少数大赚交易，还是分布均匀？
- 亏损是小额分散止损，还是存在大额单笔亏损？
- 盈亏比是否健康（≥ 2:1 为良好）

### 5. 输出复盘报告

```
## 交易复盘（{起始日期} ~ {结束日期}）

### 量化概览

| 指标 | 数值 |
|------|------|
| 交易笔数 | {n} 笔（盈 {w} / 亏 {l}） |
| 胜率 | {win_rate}% |
| 总盈亏 | {total_pnl} 元 |
| 赔率（盈亏比） | {avg_win} : {avg_loss} |
| 期望值 | {ev} 元/笔 |
| 最大单笔盈利 | {max_win} 元（{标的}） |
| 最大单笔亏损 | {max_loss} 元（{标的}） |
| 平均持有天数 | {avg_days} 天 |

### 交易明细

| Trade ID | 标的 | 方向 | 开仓价 | 平仓价 | 盈亏% | 持有天数 | 平仓原因 |
|----------|------|------|--------|--------|-------|---------|---------|
| ... | ... | ... | ... | ... | ... | ... | ... |

### 行为评价

#### 做得好的
- {具体正面行为，引用交易实例}

#### 需要改进的
- {具体问题，引用交易实例，给出改进建议}

### 改进建议
{3-5 条具体可执行的改进意见，基于本轮复盘发现的模式}
```

---

## 评价原则

- **对事不对人**：评价交易行为，不评判交易者
- **引用实例**：每条评价必须至少引用一笔具体交易作为依据
- **可执行建议**：改进意见必须具体到行动层面，不说空话
- **纵向对比**：如果有多期复盘数据，指出相比上期的进步或退步
