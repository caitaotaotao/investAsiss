# Holdings Manage

根据用户输入维护当前持仓信息，持久化为 Excel 文件。

## 核心约束

- 股票代码必须含交易所后缀（`600519_sh`、`000001_sz`）
- 所有数值字段需保持一致精度：市值保留 2 位小数，权重保留 4 位小数（如 `0.1523`）
- 每次变更持仓后必须重算全部权重，确保权重之和 = 1.0

---

## 文件规范

- **路径**：`/app/custom/trading-workspace/holdings/holdings.xlsx`
- **Sheet 名**：`holdings`

### Columns

| 列名 | 类型 | 说明                         |
|------|------|----------------------------|
| date | string | 持仓快照日期，`YYYY-MM-DD` 格式     |
| code | string | 股票代码，含交易所后缀（如 `600519_sh`） |
| short_name | string | 股票简称（如 `贵州茅台`）             |
| quantity | int | 持有数量（股）                    |
| market_value | float | 持有市值（元），= 数量 × 当前价格        |
| weight | float | 持有权重，= 该标的市值 / 账户总价值       |
| total_account_value | float | 账户总价值（元），所有持仓市值之和 + 现金     |

### 数据约束

- 同一 `date` 下，同一 `code` 只能有一行记录
- `weight` 列所有行之和应 ≤ 1.0（差额为现金占比）
- 卖出后 `quantity` 为 0 的标的从当日快照中移除

## 触发场景

| 用户意图 | 示例 | 操作 |
|---------|------|------|
| 新增持仓 | "我今天买了 500 股贵州茅台，均价 1800" | 新增一行，计算市值与权重 |
| 减仓/加仓 | "茅台加仓 200 股，价格 1785" | 更新数量与均价，重算市值权重 |
| 清仓 | "茅台全部卖出" | 从持仓中移除该标的 |
| 查询持仓 | "当前持仓是什么" | 读取并展示最新快照 |
| 更新市值 | "用今天收盘价更新持仓市值" | 需用户提供价格或搜索获取，重算全部字段 |
| 设定账户总值 | "账户总资金 100 万" | 更新 total_account_value，反推现金占比 |

## 执行流程

### 1. 解析用户输入

从用户对话中提取：
- 操作类型：买入 / 卖出 / 加仓 / 减仓 / 查询
- 股票代码（确认含交易所后缀）
- 数量
- 价格（如用户未提供，询问确认）

### 2. 读取现有持仓

读取 `/app/custom/trading-workspace/holdings/holdings.xlsx`。若文件不存在，创建新文件并初始化空表头。

### 3. 执行变更

- **买入/加仓**：新增或更新对应标的行，计算 `market_value = quantity × price`
- **卖出/减仓**：减少数量，若数量归零则移除该行
- **清仓**：移除该标的行

### 4. 重算全局字段

- 汇总所有标的 `market_value` 得到持仓总市值
- 如用户提供了 `total_account_value`，使用用户值；否则 `total_account_value = 持仓总市值 + 上次记录的现金`
- 重算每个标的的 `weight = market_value / total_account_value`
- 所有行的 `date` 更新为当日

### 5. 写入文件

将更新后的 DataFrame 写回 `/app/custom/trading-workspace/holdings/holdings.xlsx`，覆盖 `holdings` sheet。

### 6. 对话确认

在对话中输出变更摘要：

```
## 持仓更新（YYYY-MM-DD）

| 操作 | 标的 | 数量 | 市值 | 权重 |
|------|------|------|------|------|
| 买入 | 贵州茅台（600519.SH） | 500 | 900,000.00 | 15.23% |

**账户总价值**：5,910,000.00 元
**现金占比**：12.34%
```
