# Loss Reflect

对亏损交易进行结构化反思：组装完整数据包 → 确定性归因 → 三层反思 → 提炼教训。

## 核心约束

- **归因先于叙事**：必须先完成确定性四分类归因，再生成反思分析。禁止在归因前构造因果叙事
- **必须先读** `/app/custom/research-workspace/PHILOSOPHY.md` — 反思标准以投资哲学为锚
- 单次反思结果只写入 `/app/custom/reflection-workspace/lessons/cases/`，**不可直接修改 PHILOSOPHY.md**

---

## 触发场景

| 类型 | 条件 | 说明 |
|------|------|------|
| **自动触发** | trading-log 录入 pnl < 0 的交易记录 | 平仓时自动发起反思流程 |
| **手动触发** | 用户指定某笔交易或某个区间要求反思 | 可覆盖已有反思案例 |

---

## 执行流程

### Step 1: 数据包组装

每次反思**必须**先组装完整上下文数据包，缺失项须主动提示用户补充：

| 数据项 | 来源 | 必填 |
|--------|------|------|
| 交易事实 | `/app/custom/trading-workspace/trading-logs/trading_log.xlsx` | 是 |
| 研究基础 | `/app/custom/research-workspace/knowledge/stock-rationale-card/{代码}_{交易所小写}.md` | 是 |
| 交易计划 | trade-plan-review 的审查记录（如有） | 建议 |
| 市场环境 | 用户提供或通过搜索补充 | 建议 |
| 哲学锚点 | `/app/custom/research-workspace/PHILOSOPHY.md` 中与该交易逻辑相关的章节 | 是 |

**数据包组装原则**：
- 交易事实和研究基础为硬性必填，缺失则暂停反思并提示
- 市场环境如用户未提供，可简要搜索补充，但不得耗费大量时间
- 哲学锚点须精确定位到具体章节（如 §2.2 成长股、§3.1 数据验证原则），不可笼统引用全文

### Step 2: 确定性归因

在生成任何反思分析之前，先将亏损归入以下四类之一：

| 归因类型 | 代码 | 判定条件 |
|---------|------|---------|
| **研究错误** | `research_error` | 核心投资假设被证伪（如增长驱动不成立、周期判断错误、竞争格局变化未预见） |
| **执行错误** | `execution_error` | 研究正确但执行偏离计划（未按计划价格/仓位执行、追涨杀跌、提前/延迟建仓） |
| **风控错误** | `risk_error` | 风控规则未遵守（未设止损、止损未执行、单标的仓位超限、加仓已亏损头寸） |
| **市场噪声** | `market_noise` | 研究正确 + 执行正确 + 风控正确，但市场未在预期时间窗口内兑现 |

**归因决策树**：

```
1. 过程是否正确？（执行是否遵循计划）
   ├─ 否 → 进一步判断：
   │       风控规则是否被违反？
   │       ├─ 是 → risk_error
   │       └─ 否 → execution_error
   └─ 是 → 进一步判断：
           研究论证是否正确？（核心假设是否被证伪）
           ├─ 是 → market_noise
           └─ 否 → research_error
```

**归因纪律**：
- 每笔交易只标记**一个主归因**，选择因果链中最上游的错误
- 归因必须指向**具体事实**（如"产能利用率数据仅引用单一券商"），不可使用模糊表述
- 对 `market_noise` 的标记需审慎：只有研究、执行、风控三层均无明显问题时才可归为噪声

### Step 3: 三层反思

基于数据包和归因结果，从三个维度进行定性分析：

#### 3.1 研究反思（Research Reflection）

- 核心投资假设是否成立？哪些假设被证实，哪些被证伪？
- 数据验证是否充分？是否符合 PHILOSOPHY §3.1 的交叉验证要求？
- 信息源质量如何？是否过度依赖单一来源？
- 周期定位是否准确？是否存在将周期性复苏误判为趋势性成长？
- 是否存在 PHILOSOPHY §4 所列的风险盲区（估值风险、叙事陷阱等）？

#### 3.2 执行反思（Execution Reflection）

- 实际执行与交易计划的偏差有多大？
- 建仓时机是否合理？是否追高或过早入场？
- 是否存在情绪化决策的痕迹？
- 部分建仓/分批建仓策略是否合理？

#### 3.3 风控反思（Risk Reflection）

- 止损位设定是否合理？是否严格执行？
- 仓位大小是否在限制范围内？
- 是否存在在亏损头寸上加仓的行为？
- 整体组合风险敞口是否合理？

**反思纪律**：
- 各层反思必须引用数据包中的具体事实，不可空泛
- 与主归因一致的层做深度分析（主体 60%），其余两层做简要评估（各 20%）
- 归因为 `market_noise` 时，三层反思均简要说明"无明显问题"即可，重点总结从中能学到什么

### Step 4: 提炼教训

从反思中提取一条可复用的教训，包含三个要素：

| 要素 | 说明 |
|------|------|
| **一句话教训** | 简洁、可执行的规则描述 |
| **适用条件** | 该教训在什么场景下适用 |
| **反面行为** | 违反该教训的具体行为描述 |

教训须具有**可操作性**——未来遇到类似场景时，能直接据此做出不同决策。

### Step 5: 写入案例文件

将完整反思写入 `/app/custom/reflection-workspace/lessons/cases/{case_id}.md`。

**Case ID 格式**：`C{YYYYMMDD}{NNN}`，如 `C20260328001`

**文件模板**：

```markdown
---
case_id: C20260328001
trade_id: T20260328001
code: 600519.SH
short_name: 贵州茅台
attribution: research_error
date: 2026-03-28
pnl: -45000
pnl_pct: -0.05
---

# 反思案例：贵州茅台（600519.SH）

## 交易事实
开仓 2026-03-10 @ 1800.00 × 500 股 → 平仓 2026-03-28 @ 1710.00
盈亏：-45,000 元（-5.00%），持有 18 天，止损平仓

## 数据包

### 研究基础
{摘录 rationale-card 核心逻辑：周期定位、成长驱动、关键假设}

### 交易计划
{原始审查结论}

### 市场环境
{交易期间大盘/行业/风格主要变化}

### 哲学锚点
{对应 PHILOSOPHY.md 具体章节内容}

## 确定性归因

**主归因**：research_error
**归因依据**：核心假设"Q1 产能利用率拐点"未兑现，行业数据显示产能利用率
持续下行，属于研究阶段数据验证不足

## 三层反思

### 研究反思
{深度分析}

### 执行反思
{简要评估}

### 风控反思
{简要评估}

## 提炼教训

**一句话教训**：行业拐点假设必须有 ≥2 个独立数据源交叉验证，
单一券商预测不可作为唯一依据
**适用条件**：基于周期拐点判断的建仓决策
**反面行为**：仅凭单一来源的拐点预测即建仓
```

### Step 6: 对话输出

在对话中输出反思报告摘要：

```
## 亏损反思：{简称}（{代码}）

**交易**：{开仓日期} @ {开仓价} → {平仓日期} @ {平仓价}，盈亏 {pnl}（{pnl_pct}%）
**主归因**：{归因类型中文}

### 三层反思摘要
- **研究**：{1-2 句}
- **执行**：{1 句}
- **风控**：{1 句}

### 教训
{一句话教训}

> 已写入 /app/custom/reflection-workspace/lessons/cases/{case_id}.md
```

### Step 7: 触发模式检查

案例写入后，自动检查同类 attribution 的累计案例数：
- 若同类 ≥ 3 → 在对话中提示"已检测到同类错误模式，建议运行 pattern-detect"
- 若 < 3 → 不提示，静默完成

## 与其他技能的协同

- **trading-log** → 提供交易事实数据
- **pattern-detect** → loss-reflect 写入新 case 后可能触发模式检测
- **trade-review** → 区间复盘中识别的亏损交易可批量触发 loss-reflect
